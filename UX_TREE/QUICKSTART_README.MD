What you're seeing:
The main spine runs vertically down the left side as a soft blue pill-shaped bar that fills downward as overall progress increases. Each main step (understanding your vision, setting up project, building pages, creating images, payments, testing, publishing) sits on the spine with a dot indicator.
Click any step with children and it cascades open — "Building your pages" expands to show homepage layout, navigation, product gallery, footer. Product gallery itself has sub-branches: grid layout, product cards, filter bar. The branches are thinner, softer hue, forking off to the right with horizontal connector bars — exactly like your image shows.
"Creating brand images" has a parallel tag because it's running simultaneously alongside "Building your pages" — two branches off the main spine at the same time.
The query widget drops inline in the tree at the decision point — not a modal, not a popup. Three soft-bordered pill buttons (Minimal / Bold / Playful) with a timeout countdown. User picks one, it routes to the specific agent and disappears.
At the bottom: the output node (your finished site — live URL, project files, performance report), pause/cancel controls, and the text input bar for mid-build corrections. "Your feedback is routed to the right step — other work continues."
What this connects to in the specs:
Every StatusLine from the Status Agent translator map feeds a tree node. Every UserQuery renders as the inline button widget. Every interrupt from the text input goes through the Status Agent → PA surgical routing pipeline. The progress bar filling the spine is calculateProgress() from the Status Agent spec. The cascading open/close is the expandable flag on StatusLine. The "parallel" tag is set when the Conductor's DAG has concurrent steps with no dependency between them.
The colour palette is soft blue-white watercolour — #D6E4F7 for pending spine, #7BAAF0 for active, #A8C4E8 for done. Pill radius everywhere. No sharp edges. No terminal energy. This is the showroom, not the engine room.
