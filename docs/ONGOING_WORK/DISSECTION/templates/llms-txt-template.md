# llms.txt Template
<!-- triggers: llms, llms.txt, ai-friendly, machine-readable, seo -->
<!-- version: 1.0 -->
<!-- source: custom -->
<!-- runtime: any -->

## Purpose

Generate llms.txt after ANY site build. This zero-cost feature makes deployed sites AI-friendly, allowing LLM assistants to understand the site when users ask about it.

## Why It Matters

- Users share their site URLs with AI assistants
- AI assistants read llms.txt to understand what the site does
- Better AI understanding = better assistance for the user
- Zero extra LLM calls (pure string templating from existing data)

## Template: llms.txt (Short Version)

```
# {project_name}

> {one_line_description}

## About

{what_the_business_does}

## Pages

{list_of_routes_built}

## Tech

Built with {stack}. Deployed at {url}.

## Contact

{contact_info_if_available}
```

## Template: llms-full.txt (Extended Version)

```
# {project_name}

> {one_line_description}

## About

{what_the_business_does}

## Features

{bulleted_list_of_features}

## Pages

### {page_1_name}
- Route: {route}
- Purpose: {what_this_page_does}
- Key elements: {main_components}

### {page_2_name}
- Route: {route}
- Purpose: {what_this_page_does}
- Key elements: {main_components}

## Tech Stack

- Frontend: {framework}
- Styling: {css_solution}
- Backend: {if_applicable}
- Database: {if_applicable}
- Deployment: {platform}

## API

{if_api_exists}
- Base URL: {base_url}
- Authentication: {auth_method}
- Endpoints: {list_of_endpoints}

## Contact

{contact_info}

---

Generated by Unified Terminal on {date}
```

## Data Sources

Pull from existing artifacts:
- `project_name` → Intake brief or PROJECT.md
- `one_line_description` → Intake brief
- `what_the_business_does` → Intake brief or CONTEXT.md
- `list_of_routes_built` → Route scan or package.json scripts
- `stack` → STACK.md or package.json
- `url` → Deploy agent output

## Generation Process

1. Read PROJECT-ARCHIVE.md (if exists) or Spine data
2. Extract fields into template
3. Write llms.txt to project root
4. Optionally write llms-full.txt for detailed version
5. Notify user: "Created llms.txt so AI assistants can understand your site"

## Hard Boundaries

- NEVER skip llms.txt generation after successful deploy
- NEVER include sensitive data (API keys, internal URLs)
- ALWAYS use data from existing artifacts (no extra LLM calls)

## Success Looks Like

- [ ] llms.txt exists in project root
- [ ] llms.txt is < 500 bytes (short version) or < 2KB (full version)
- [ ] All template fields populated (or marked as N/A)
- [ ] No sensitive information included
- [ ] File is valid UTF-8 text
